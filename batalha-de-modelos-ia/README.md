# ⚔️ Batalha de Modelos & Engenharia de Prompt (XML)

## 📝 Descrição do Projeto
Este projeto consiste em um estudo comparativo aprofundado utilizando técnicas de **Engenharia de Prompt** estruturadas em **XML**. O objetivo foi desafiar diferentes modelos de linguagem (LLMs) a gerarem uma plataforma SaaS completa chamada **NightHub** — um sistema voltado para a gestão de eventos e promoções de adegas e tabacarias.

> 📄 **Análise Completa (PDF):** [Baixar Relatório de Comparação](./Batalha-de-Modelos-&-Engenharia-de-Prompt.pdf)

---

## 🛠️ O Desafio Técnico
Foi desenvolvido um prompt estruturado para gerar uma **Single Page Application (SPA)** com:
* **Design:** Estética *Dark Mode* com paleta de cores Preto, Roxo e Neon.
* **Funcionalidades:** Sistema de login, feed de atualizações em tempo real, geolocalização e notificações push.
* **Arquitetura:** HTML5 e CSS3 integrado em arquivo único.

## 📊 Ferramentas Testadas
A performance foi avaliada entre os modelos:
* **OpenAI:** GPT
* **Google:** Gemini
* **Anthropic:** Claude
* **DeepSeek, Qwen, Grok & Maritaca**

## 🏆 Resultados da Análise
Com base nos testes realizados:
* **Melhor Compreensão:** O modelo **GROK** se destacou na formulação dos requisitos de interface e design.
* **Melhor para Códigos Complexos:** O **CLAUDE** apresentou o desempenho mais robusto para recursos avançados.
* **Prototipagem Rápida:** O **DEEPSEEK** foi o mais eficaz na entrega minimalista e funcional.
* **Variação de Tokens:** Observou-se uma oscilação significativa de **900 a 23.000 tokens** para resultados similares.

## 🧠 Reflexão Crítica
O projeto demonstrou que a precisão do resultado está diretamente ligada à estruturação do prompt (XML) e que ferramentas com maior processamento de tokens tendem a entregar modelos mais complexos, embora a assistência humana continue sendo vital para ajustes finos.
